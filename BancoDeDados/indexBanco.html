3.1 INTRODUÇÃO AO CAPÍTULO 3

Esta seção apresenta a camada de persistência modelada para o sistema de controle e monitoramento de acesso dos computadores, como também de usuários em geral, sendo eles: alunos, professores, servidores e administradores. Especificamente, o administrador é a pessoa responsável pelo sistema de gerenciamento de banco de dados. Utilizamos o software MySQL, em linguagem SQL (Linguagem de Consulta Estruturada) para criação do SGBD (Sistema de Gerenciamento de Banco de Dados) , juntamente com o ambiente de desenvolvimento integrado Eclipse IDE for Java Developers, onde é desenvolvido o código na linguagem de programação em Java.

3.2 CRIAÇÃO DO BANCO DE DADOS

Tendo em vista que o objetivo da criação do banco de dados é controlar o acesso a quem utiliza os computadores, foi desenvolvido este SGBD (Sistema de Gerenciamento de Banco de Dados). Portanto, algumas etapas foram necessárias, estas serão devidamente explicitadas e detalhadas, para uma melhor compreensão do que foi proposto e realizado.

Inicialmente, foi criada uma tabela no banco de dados, onde é definido os campos considerados importantes para serem armazenados, como: matrícula, nome, CPF, e-mail, etc.

3.3 DESENVOLVIMENTO DO CÓDIGO JAVA

Primeiramente, foi feita a conexão entre os arquivos .java e o SGBD, como também a estruturação dos packages (pacotes) no Eclipse, de forma que ficasse da seguinte maneira:

3.3.1 ModeloDAO

É onde tem a conexão anteriormente citada e o arquivo que contém as funções relacionadas ao SGBD, como, por exemplo, a de inserir um novo usuário. Foi adicionado “DAO” ao final do nome da package, a fim de identificar o local em que se encontram os arquivos .class com as Strings sql, as quais farão a modificação nas tabelas banco de dados.

3.3.2 ModeloEntidade

Onde está contida a classe usuário, para sempre que for preciso criar um objeto, este seja do tipo Usuário.

3.3.3 Visão

Seguindo adiante utilizando o software Eclipse IDE, dentro do package visão, há o terminal e nele inicia-se o código com uma função capaz de oferecer um menu, onde é listado as operações que o SGBD torna-se capaz de realizar no banco de dados.

Dessa forma, cada opção do menu possui um bloco de código responsável por fazê-la funcionar. O objetivo desse programa, além do gerenciamento de acesso aos usuários, é proporcionar uma boa experiência a quem está utilizando.

3.3.4 AbstratoDAO

Essa classe realiza a comunicação entre o banco de dados e o Eclipse IDE, e é por causa dele que conseguimos alocar os dados, remover ou atualizar os dados. É necessário informar o hostname, o nome, user e senha do banco de dados para realizar uma conexão efetiva. Percebe-se isso na linha 14 e 15 do banco de dados.

3.3.5 DateUtil

A classe java.util.Date representa e fornece construtores e métodos para lidar com data e hora em java. Vimos ela no código quando cadastramos ou atualizamos os dados quando era pedido a data de nascimento.
